## はじめに
Vue で Scss と Sass でメンテされているプロジェクトがあります
これを Lightning CSS に移行します

Vue の scoped style を活用する前提ですが、最新の CSS をダウンコンパイルする機能を活用してメンテナビリティを極限まで高めた移行を行いたいです

Scss, Lightning CSS を併用することはできないのでどこかで一括で入れ替える必要があります
しかし、これは実装コストばかりかレビューコストが嵩み、さらに現代では AI Agent のコンテキスト長も膨大になり品質に影響を及ぼします

そこで段階的な意向戦略を取る必要があります
大まかにデザインシステムとアプリのモノリポ構成のようになっているプロジェクトです
いくつかの歴史的事情により制約があります
ざっくり言うとデザインシステム側にはビルドプロセスが存在しません
デザインシステム側のコードはおおよそ `cp` コマンドではないですがアプリ側にコピーされます
そのため、モノリポ構成とはなっていますが、どちらから先にビルドプロセスを移行するというような戦略は取ることができません

そこで現在考えているのは `<style lang="scss">` や `.scss` ファイルのまま
